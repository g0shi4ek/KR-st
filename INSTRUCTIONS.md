# Вариант 11 — Передача текстовых файлов по RS232C

---

## ⚡ БЫСТРЫЙ СТАРТ — Запуск и демонстрация за 5 шагов

> Этот раздел — для быстрой демонстрации на **одном ПК** с виртуальными COM-портами
> (com0com) или на **двух ПК** с нульмодемным кабелем.

### Вариант А: Один ПК + виртуальные COM-порты (com0com)

#### Шаг 1 — Установить com0com и создать пару портов

1. Скачать **com0com** с https://sourceforge.net/projects/com0com/
2. Установить (требуются права администратора)
3. Открыть **Setup Command Prompt** (из меню Пуск → com0com)
4. Выполнить команду:
   ```
   install PortName=COM1 PortName=COM2
   ```
5. Убедиться, что в **Диспетчере устройств** появились `COM1` и `COM2` в разделе «Порты»

#### Шаг 2 — Собрать проект

Открыть `KR.sln` в Visual Studio 2022 → **Ctrl+Shift+B**

Или через командную строку (если установлен .NET 8 SDK):
```cmd
dotnet build KR_KSN.csproj
```

Исполняемый файл: `bin\Debug\net8.0-windows\KR_KSN.exe`

#### Шаг 3 — Запустить ПРИЁМНИК (первым!)

1. Открыть папку `bin\Debug\net8.0-windows\`
2. Запустить `KR_KSN.exe` — **первый экземпляр**
3. В окне «Выбор роли» нажать **«Приёмник (Receiver, COM2)»**
4. В главном окне нажать **«Подключить»**
5. В журнале появится: `Порт COM2 открыт. Отправка UPLINK...`
6. Статус: **«Не подключено»** (ожидает Источник)

#### Шаг 4 — Запустить ИСТОЧНИК (вторым!)

1. Запустить ещё один `KR_KSN.exe` — **второй экземпляр**
2. В окне «Выбор роли» нажать **«Источник (Sender, COM1)»**
3. В главном окне нажать **«Подключить»**
4. В журнале обоих окон появится:
   ```
   → UPLINK
   ← ACK_UPLINK
   ```
5. Статус в обоих окнах изменится на **«Подключено»** (зелёный)

#### Шаг 5 — Отправить файл

**На стороне Источника:**
1. Нажать **«Обзор...»** → выбрать любой `.txt` файл
   (можно использовать `Виктория_System.txt` из папки проекта)
2. Содержимое файла отобразится в панели «Просмотр содержимого файла»
3. (Опционально) Выбрать темп: **«Медленно»** для наглядной демонстрации
4. Нажать **«Задать темп»**
5. Нажать **«Отправить файл»**

**Что происходит (наблюдать в журнале):**
```
→ FILE_INFO "Виктория_System.txt" (3 chunks)
← ACK_FILE_INFO
→ CHUNK #0 (128 bytes, attempt 1)
← ACK_CHUNK #0
→ CHUNK #1 (128 bytes, attempt 1)
← ACK_CHUNK #1
→ CHUNK #2 (44 bytes, attempt 1)
← ACK_CHUNK #2
→ TRANSFER_DONE
← ACK_DONE
```

**На стороне Приёмника:**
- Содержимое файла появляется в реальном времени в панели «Просмотр»
- Прогресс-бар заполняется по мере приёма блоков
- По завершении появляется сообщение «Передача завершена»
- Нажать **«Сохранить»** → выбрать путь → файл сохранён на диск

### Вариант Б: Два ПК + нульмодемный кабель

#### Подготовка кабеля

Нульмодемный кабель DB9 ↔ DB9 (минимальная распайка):
```
ПК-1 (COM1)          ПК-2 (COM2)
  TXD (2) ──────────── RXD (3)
  RXD (3) ──────────── TXD (2)
  GND (5) ──────────── GND (5)
  RTS (7) ──────────── CTS (8)
  CTS (8) ──────────── RTS (7)
  DTR (4) ──────────── DSR (6)
  DSR (6) ──────────── DTR (4)
```

#### Запуск

1. Скопировать папку `bin\Debug\net8.0-windows\` на оба ПК
2. На **ПК-2** запустить `KR_KSN.exe` → выбрать **«Приёмник»** → нажать **«Подключить»**
3. На **ПК-1** запустить `KR_KSN.exe` → выбрать **«Источник»** → нажать **«Подключить»**
4. Дождаться статуса **«Подключено»** на обоих ПК
5. На ПК-1 выбрать файл и нажать **«Отправить файл»**

---

### Демонстрация управления темпом (прикладной уровень)

Во время передачи файла (особенно наглядно при «Медленном» темпе):

**На стороне Приёмника:**
1. Выбрать в списке «Управление темпом» → **«Быстро (10 мс)»**
2. Нажать **«Задать темп»**
3. В журнале Источника появится: `← PACE_FAST`, `Темп → FAST (10 мс)`
4. Передача ускорится

**На стороне Источника:**
1. Выбрать в списке «Темп передачи» → **«Медленно (500 мс)»**
2. Нажать **«Задать темп»**
3. В журнале Приёмника появится: `← PACE_SLOW`, `Темп → SLOW (500 мс)`

---

### Демонстрация кода Хэмминга

Код Хэмминга работает автоматически при каждой передаче:
- Каждый байт файла кодируется в 2 байта (14 информационных бит + 2 нулевых)
- При одиночной ошибке бит исправляется автоматически (без повтора блока)
- При двойной ошибке блок запрашивается повторно (RET_CHUNK)

В журнале при успешном декодировании:
```
CHUNK #0 OK (128 bytes)
```

При ошибке и исправлении (одиночный сбой):
```
CHUNK #0 OK (128 bytes)   ← ошибка исправлена кодом Хэмминга, ACK отправлен
```

При неисправимой ошибке:
```
CHUNK #0 decode error → RET   ← запрос повтора
→ RET_CHUNK #0, retrying...
```

## Описание задания

Разработан протокол взаимодействия объектов до прикладного уровня локальной сети,
состоящей из **2-х ПК**, соединённых через интерфейс **RS232C нульмодемным кабелем**.
Реализована функция **передачи текстовых файлов** с просмотром содержимого при приёме
и управлением темпом передачи на прикладном уровне.

Скорость обмена и параметры COM-порта **заданы по умолчанию** (9600 bps, 8 бит данных,
без чётности, 1 стоп-бит). Файл задаётся источником. При передаче файла информация
защищается **[7,4]-кодом Хэмминга** с исправлением одиночных ошибок.

### Структура кадра

```
[0xFF] [FrameType] [payload...]
```

| FrameType      | Код  | Описание                                      |
|----------------|------|-----------------------------------------------|
| UPLINK         | 0x01 | Запрос соединения (Источник → Приёмник)       |
| ACK_UPLINK     | 0x02 | Соединение принято                            |
| RET_UPLINK     | 0x03 | Соединение отклонено                          |
| DOWNLINK       | 0x04 | Запрос разрыва соединения                     |
| ACK_DOWNLINK   | 0x05 | Разрыв подтверждён                            |
| FILE_INFO      | 0x10 | Метаданные файла: имя + кол-во блоков         |
| ACK_FILE_INFO  | 0x11 | Приёмник готов принять файл                   |
| RET_FILE_INFO  | 0x12 | Ошибка метаданных                             |
| CHUNK          | 0x20 | Блок данных (Хэмминг-кодированный)            |
| ACK_CHUNK      | 0x21 | Блок принят успешно                           |
| RET_CHUNK      | 0x22 | Ошибка блока, повторить                       |
| PACE_SLOW      | 0x30 | Управление темпом: медленно (500 мс/блок)     |
| PACE_NORMAL    | 0x31 | Управление темпом: нормально (100 мс/блок)    |
| PACE_FAST      | 0x32 | Управление темпом: быстро (10 мс/блок)        |
| TRANSFER_DONE  | 0x40 | Все блоки отправлены                          |
| ACK_DONE       | 0x41 | Передача завершена (подтверждение)            |

### [7,4]-код Хэмминга

Каждый байт файла разбивается на два 4-битных полубайта.
Каждый полубайт кодируется в 7 бит по схеме Хэмминга [7,4]:

```
Позиции (0-based, LSB first):
  0=d1, 1=d2, 2=d3, 4=d4  — информационные биты
  3=p1, 5=p2, 6=p3         — контрольные биты

p1 = d1 ⊕ d2 ⊕ d3
p2 = d1 ⊕ d2 ⊕ d4
p3 = d1 ⊕ d3 ⊕ d4
```

При декодировании вычисляется синдром (3 бита). Если синдром ≠ 0,
**одиночная ошибка исправляется** автоматически.

## Требования к оборудованию

- **2 компьютера** под управлением Windows 10/11
- **Нульмодемный кабель RS232C** (DB9 ↔ DB9)
  - Распайка нульмодема: TXD(2)↔RXD(3), RXD(3)↔TXD(2), GND(5)↔GND(5), RTS(7)↔CTS(8), CTS(8)↔RTS(7)
- На каждом ПК должен быть доступен COM-порт (физический или USB-to-COM адаптер)
- **.NET 8 Runtime** (Windows Desktop) установлен на обоих ПК

## Запуск программы

> **Важно:** программу нужно запустить на **обоих компьютерах** одновременно.
> Сначала запускайте **Приёмник**, затем **Источник**.

### Шаг 1 — Подключить нульмодемный кабель

Соедините COM1 первого ПК с COM2 второго ПК нульмодемным кабелем.

> Если используются USB-to-COM адаптеры, убедитесь что в Диспетчере устройств
> они определились как **COM1** и **COM2** соответственно.
> При необходимости переименуйте порты в свойствах устройства.

### Шаг 2 — Запустить программу на ПК-Приёмнике (COM2)

1. Запустить `KR_KSN.exe`
2. В окне выбора роли нажать **«Приёмник (Receiver, COM2)»**
3. Нажать кнопку **«Подключить»** — программа откроет COM2 и будет ожидать входящего соединения

### Шаг 3 — Запустить программу на ПК-Источнике (COM1)

1. Запустить `KR_KSN.exe`
2. В окне выбора роли нажать **«Источник (Sender, COM1)»**
3. Нажать кнопку **«Подключить»** — программа откроет COM1 и отправит кадр UPLINK

### Шаг 4 — Установка соединения

- Источник отправляет `UPLINK`
- Приёмник отвечает `ACK_UPLINK`
- В строке состояния обоих окон появится **«Подключено»** (зелёный цвет)

## Отправка файла (на ПК-Источнике)

1. Нажать кнопку **«Обзор...»** и выбрать текстовый файл (`.txt` или любой другой)
2. Содержимое файла отобразится в панели **«Просмотр содержимого файла»**
3. При необходимости задать темп передачи:
   - Выбрать в выпадающем списке **«Темп передачи»**: Медленно / Нормально / Быстро
   - Нажать **«Задать темп»** — кадр `PACE_*` будет отправлен Приёмнику
4. Нажать **«Отправить файл»**

### Процесс передачи

```
Источник                          Приёмник
   │                                  │
   │──── FILE_INFO ──────────────────►│  (имя файла, кол-во блоков)
   │◄─── ACK_FILE_INFO ───────────────│
   │                                  │
   │──── CHUNK #0 (Хэмминг) ────────►│
   │◄─── ACK_CHUNK ──────────────────│
   │                                  │
   │──── CHUNK #1 (Хэмминг) ────────►│
   │◄─── ACK_CHUNK ──────────────────│  (или RET_CHUNK при ошибке → повтор)
   │         ...                      │
   │──── TRANSFER_DONE ─────────────►│
   │◄─── ACK_DONE ───────────────────│
```

- Прогресс-бар показывает процент отправленных блоков
- При ошибке блок повторяется до 3 раз
- Размер блока: **128 байт** (до кодирования Хэммингом)

---

## Приём файла (на ПК-Приёмнике)

1. После установки соединения Приёмник автоматически ожидает `FILE_INFO`
2. При получении первого кадра `FILE_INFO` в поле **«Принятый файл»** появится имя файла
3. Содержимое файла отображается **в реальном времени** в панели **«Просмотр содержимого файла»** по мере поступления блоков
4. Прогресс-бар показывает процент принятых блоков

### Управление темпом на стороне Приёмника

Приёмник может в любой момент запросить изменение темпа передачи:
1. Выбрать нужный темп в выпадающем списке **«Управление темпом»**
2. Нажать **«Задать темп»** — кадр `PACE_SLOW` / `PACE_NORMAL` / `PACE_FAST` отправляется Источнику
3. Источник получает кадр и меняет задержку между блоками

### Сохранение принятого файла

1. После завершения передачи появится сообщение **«Передача завершена»**
2. Нажать кнопку **«Сохранить»**
3. Выбрать путь и имя файла в диалоге сохранения
4. Файл будет записан на диск

## Отключение

1. Нажать кнопку **«Отключить»** на любом из ПК
2. Будет отправлен кадр `DOWNLINK`, другая сторона ответит `ACK_DOWNLINK`
3. COM-порт закроется, статус изменится на **«Не подключено»**

## Тестирование без физического оборудования

Для тестирования на одном ПК можно использовать **виртуальные COM-порты**:

1. Установить **com0com** (https://sourceforge.net/projects/com0com/)
2. Создать пару виртуальных портов: **COM1 ↔ COM2**
3. Запустить два экземпляра `KR_KSN.exe` на одном ПК:
   - Первый экземпляр → роль **Источник** (COM1)
   - Второй экземпляр → роль **Приёмник** (COM2)

## Возможные ошибки и решения

| Ошибка | Причина | Решение |
|--------|---------|---------|
| «Не удалось открыть порт COM1/COM2» | Порт занят другой программой или не существует | Закрыть другие программы, использующие COM-порт; проверить Диспетчер устройств |
| Таймаут при ожидании ACK_CHUNK | Кабель не подключён или неправильная распайка | Проверить кабель и распайку нульмодема |
| Блок повторяется 3 раза и передача прерывается | Помехи на линии | Уменьшить темп передачи (выбрать «Медленно») |
| Файл сохраняется с искажениями | Ошибки, превышающие возможности кода Хэмминга (>1 бит/блок) | Уменьшить скорость передачи или проверить кабель |
