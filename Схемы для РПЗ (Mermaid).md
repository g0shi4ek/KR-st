# Схемы для РПЗ — Вариант 11

> **Как получить изображения схем:**
> 1. Зайди на **https://mermaid.live**
> 2. Вставь код нужной схемы в левое поле
> 3. Справа появится готовая схема — нажми **PNG** или **SVG** → скачай → вставь в Word
>
> Каждая схема оформляется на отдельном листе формата А4 с угловым штампом МГТУ.

## ЛИСТ 2. ВРЕМЕННЫЕ ДИАГРАММЫ

### 2а. Успешное установление соединения

```mermaid
sequenceDiagram
    participant A as ПК-1
    participant B as ПК-2

    Note over A: Нажата кнопка «Подключиться»
    A->>B: UPLINK-кадр [0xFF][0x01]
    B->>A: ACK_UPLINK-кадр [0xFF][0x02]
    Note over A,B: Логическое соединение установлено
```

### 2б. Неуспешное установление соединения (3 таймаута)

```mermaid
sequenceDiagram
    participant A as ПК-1
    participant B as ПК-2

    A->>B: UPLINK-кадр
    Note over A: Таймаут 1 (3000 мс)
    A->>B: UPLINK-кадр (повтор 1)
    Note over A: Таймаут 2 (3000 мс)
    A->>B: UPLINK-кадр (повтор 2)
    Note over A: Таймаут 3 (3000 мс)
    Note over A: Соединение не установлено — ошибка
```

### 2в. Успешная передача файла

```mermaid
sequenceDiagram
    participant A as ПК-1 (Отправитель)
    participant B as ПК-2 (Получатель)

    A->>B: FILE_INFO-кадр [0xFF][0x10] (имя, N блоков)
    B->>A: ACK_FILE_INFO [0xFF][0x11]

    A->>B: CHUNK[0]-кадр [0xFF][0x20]
    B->>A: ACK_CHUNK [0xFF][0x21]
    Note over A: Задержка темпа (10/100/500 мс)

    A->>B: CHUNK[1]-кадр [0xFF][0x20]
    B->>A: ACK_CHUNK [0xFF][0x21]
    Note over A: Задержка темпа

    Note over A,B: ... (блоки 2..N-2) ...

    A->>B: CHUNK[N-1]-кадр [0xFF][0x20]
    B->>A: ACK_CHUNK [0xFF][0x21]

    A->>B: TRANSFER_DONE [0xFF][0x40]
    B->>A: ACK_DONE [0xFF][0x41]
    Note over A,B: Передача завершена
```

### 2г. Передача с ошибкой и повторной отправкой

```mermaid
sequenceDiagram
    participant A as ПК-1 (Отправитель)
    participant B as ПК-2 (Получатель)

    A->>B: CHUNK[N]-кадр [0xFF][0x20]
    Note over B: Ошибка декодирования Хэмминга
    B->>A: RET_CHUNK [0xFF][0x22]
    A->>B: CHUNK[N]-кадр (повтор) [0xFF][0x20]
    B->>A: ACK_CHUNK [0xFF][0x21]
    Note over A: Задержка темпа
    A->>B: CHUNK[N+1]-кадр [0xFF][0x20]
```

### 2д. Управление темпом передачи

```mermaid
sequenceDiagram
    participant A as ПК-1 (Отправитель)
    participant B as ПК-2 (Получатель)

    Note over A: Темп: NORMAL (100 мс)
    A->>B: CHUNK[i]-кадр
    B->>A: ACK_CHUNK
    B->>A: PACE_SLOW [0xFF][0x30]
    Note over A: Темп изменён: SLOW (500 мс)
    A->>B: CHUNK[i+1]-кадр
    B->>A: ACK_CHUNK
    B->>A: PACE_FAST [0xFF][0x32]
    Note over A: Темп изменён: FAST (10 мс)
    A->>B: CHUNK[i+2]-кадр
    B->>A: ACK_CHUNK
```

### 2е. Успешное разъединение

```mermaid
sequenceDiagram
    participant A as ПК-1 (Отправитель)
    participant B as ПК-2 (Получатель)

    Note over A: Нажата кнопка «Отключиться»
    A->>B: DOWNLINK-кадр [0xFF][0x04]
    B->>A: ACK_DOWNLINK-кадр [0xFF][0x05]
    Note over A,B: Логическое соединение разорвано
    Note over A: Порт COM1 закрыт
    Note over B: Порт COM2 закрыт
```

### Информационный кадр канального уровня (FILE_INFO-кадр)

```
┌──────────┬──────────┬──────────────────┬──────────────────────┬──────────────────┐
│11111111  │00010000  │  name_len         │  Имя файла           │  total_chunks    │
│  1 байт  │  1 байт  │  2 байта (LE)    │  name_len байт       │  4 байта (LE)    │
└──────────┴──────────┴──────────────────┴──────────────────────┴──────────────────┘
Старт-байт  Тип кадра  Длина имени файла  Имя файла (UTF-8)      Кол-во блоков
```

### Информационный кадр канального уровня (CHUNK-кадр)

```
┌──────────┬──────────┬──────────────────┬──────────────────┬──────────────────────────┐
│11111111  │00100000  │  chunk_index      │  data_len        │  Данные (Хэмминг)        │
│  1 байт  │  1 байт  │  4 байта (LE)    │  2 байта (LE)    │  до 256 байт             │
└──────────┴──────────┴──────────────────┴──────────────────┴──────────────────────────┘
Старт-байт  Тип кадра  Номер блока        Длина данных       Закодированные данные [7,4]
```

### Блок данных прикладного уровня

```
┌─────────────────────────────────────────────────────────────────────────────────────┐
│  Содержимое текстового файла (декодированные блоки, собранные в порядке chunk_index) │
└─────────────────────────────────────────────────────────────────────────────────────┘
```

## ЛИСТ 5. ГРАФ ДИАЛОГА ПОЛЬЗОВАТЕЛЯ

```mermaid
stateDiagram-v2
    direction LR

    [*] --> Форма_выбора_роли : Запуск KR_KSN.exe

    Форма_выбора_роли --> Главная_форма_Отправитель : 1. Кнопка «Отправитель»
    Форма_выбора_роли --> Главная_форма_Получатель : 2. Кнопка «Получатель»
    Форма_выбора_роли --> [*] : 3. Кнопка «Выход»

    state "Главная форма (Отправитель)" as Главная_форма_Отправитель {
        direction TB
        Ожидание_подключения_S : 1. Подключиться
        Подключён_S : 2. Отключиться
        Файл_выбран : 3. Обзор (выбор файла)
        Передача : 4. Отправить файл
        Темп_S : 5. Медленно / Нормально / Быстро

        Ожидание_подключения_S --> Подключён_S : Открыть COM1\nОтправить UPLINK
        Подключён_S --> Файл_выбран : Выбрать .txt файл
        Файл_выбран --> Передача : Начать передачу
        Передача --> Темп_S : Изменить темп
        Темп_S --> Передача : Продолжение
        Передача --> Подключён_S : Передача завершена
        Подключён_S --> Ожидание_подключения_S : Отключиться
    }

    state "Главная форма (Получатель)" as Главная_форма_Получатель {
        direction TB
        Ожидание_подключения_R : 1. Подключиться
        Подключён_R : 2. Отключиться
        Приём : 3. Приём файла (авто)
        Темп_R : 4. Медленно / Нормально / Быстро
        Сохранение : 5. Сохранить файл

        Ожидание_подключения_R --> Подключён_R : Открыть COM2
        Подключён_R --> Приём : Получен FILE_INFO
        Приём --> Темп_R : Изменить темп
        Темп_R --> Приём : Продолжение
        Приём --> Сохранение : Получен TRANSFER_DONE
        Сохранение --> Подключён_R : Файл сохранён
        Подключён_R --> Ожидание_подключения_R : Отключиться
    }

    Главная_форма_Отправитель --> [*] : Закрытие окна
    Главная_форма_Получатель --> [*] : Закрытие окна
```

## ЛИСТ 6. АЛГОРИТМЫ ПРОГРАММЫ

### 6а. Открытие COM-порта

```mermaid
flowchart TD
    A([Начало]) --> B[Получение дескриптора порта\nPortSetup.OpenPort portName]
    B --> C{Получение\nудачно?}
    C -- нет --> D[Конец]
    C -- да --> E[Установка параметров порта:\nBaudRate=9600, DataBits=8\nParity=None, StopBits=One\nRTS=true, DTR=true]
    E --> F{Установка\nудачна?}
    F -- нет --> D
    F -- да --> G[Установка тайм-аутов\nReadTimeout=3000 мс\nWriteTimeout=3000 мс]
    G --> H[isOpen = true]
    H --> I([Конец])

    style A fill:#5ba85a,color:#fff
```

### 6г. Отправка кадра (SendChunk)

```mermaid
flowchart TD
    A([Начало]) --> B[Нажатие кнопки «Отправить файл»]
    B --> C{isOpen == true?}
    C -- нет --> D([Конец])
    C -- да --> E[Получить данные блока из файла]
    E --> F[Кодировать блок кодом Хэмминга 7,4\n128 байт → 256 байт]
    F --> G[Формирование CHUNK-кадра]
    G --> H[Запомнить последний отправленный кадр]
    H --> I[Отправка данных в COM-порт]
    I --> J[Включение таймера 5000 мс]
    J --> K{Получен\nACK_CHUNK?}
```

## Таблица типов кадров (для вставки в РПЗ)

| Тип кадра | Код (hex) | Код (bin) | Направление | Описание |
|---|---|---|---|---|
| UPLINK | 0x01 | 00000001 | S → R | Запрос установления соединения |
| ACK_UPLINK | 0x02 | 00000010 | R → S | Подтверждение соединения |
| RET_UPLINK | 0x03 | 00000011 | R → S | Отказ в соединении |
| DOWNLINK | 0x04 | 00000100 | S → R | Запрос разрыва соединения |
| ACK_DOWNLINK | 0x05 | 00000101 | R → S | Подтверждение разрыва |
| FILE_INFO | 0x10 | 00010000 | S → R | Метаданные файла (имя, кол-во блоков) |
| ACK_FILE_INFO | 0x11 | 00010001 | R → S | Подтверждение метаданных |
| RET_FILE_INFO | 0x12 | 00010010 | R → S | Запрос повтора метаданных |
| CHUNK | 0x20 | 00100000 | S → R | Блок данных файла (Хэмминг [7,4]) |
| ACK_CHUNK | 0x21 | 00100001 | R → S | Подтверждение блока |
| RET_CHUNK | 0x22 | 00100010 | R → S | Запрос повтора блока |
| PACE_SLOW | 0x30 | 00110000 | S ↔ R | Установить темп: 500 мс |
| PACE_NORMAL | 0x31 | 00110001 | S ↔ R | Установить темп: 100 мс |
| PACE_FAST | 0x32 | 00110010 | S ↔ R | Установить темп: 10 мс |
| TRANSFER_DONE | 0x40 | 01000000 | S → R | Передача завершена |
| ACK_DONE | 0x41 | 01000001 | R → S | Подтверждение завершения |

> Стартовый байт для всех кадров: **0xFF (11111111)**
