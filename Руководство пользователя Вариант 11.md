## 1. Назначение программы

Программа «Передача файлов» предназначена для передачи текстовых файлов между двумя компьютерами, соединёнными через интерфейс RS-232C нульмодемным кабелем. Программа реализует функцию просмотра содержимого файла в процессе приёма, а также управление темпом передачи на прикладном уровне. Передаваемые данные защищаются [7,4]-кодом Хэмминга с исправлением одиночных ошибок.

---

## 2. Условия выполнения программы

Для нормальной работы программы требуется два компьютера с процессором Intel I3 530 или выше, 2 ГБ оперативной памяти, соединённые нульмодемным кабелем через интерфейс RS-232C. Также требуется оператор (пользователь), имеющий базовые знания о работе с компьютером и операционной системой Windows 10 и выше.

Параметры COM-порта заданы по умолчанию и не требуют ручной настройки:

| Параметр        | Значение |
|-----------------|----------|
| Скорость        | 9600 bps |
| Биты данных     | 8        |
| Чётность        | None     |
| Стоп-биты       | 1        |

Для тестирования на одном компьютере требуется программа **com0com** (Virtual Serial Port Emulator) с установленной парой виртуальных портов COM1–COM2.

## 3. Выполнение программы

### 3.1. Инсталляция / деинсталляция

Инсталляция программы заключается в копировании файла `KR_KSN.exe` и всех файлов из папки `bin\Debug\net8.0-windows\` на рабочую машину. Для деинсталляции нужно удалить эту папку.

Для работы программы необходимо наличие **.NET 8 Desktop Runtime** (Windows). Скачать можно с официального сайта Microsoft: https://dotnet.microsoft.com/download/dotnet/8.0

---

### 3.2. Запуск и настройка программы com0com (для тестирования на одном ПК)

Для тестирования без физического оборудования необходимо установить программу **com0com**:

1. Скачать com0com с сайта: https://sourceforge.net/projects/com0com/
2. Выполнить стандартную установку (требуются права администратора).
3. Открыть **Setup Command Prompt** из меню «Пуск» → com0com.
4. Выполнить команду для создания пары виртуальных портов:
   ```
   install PortName=COM1 PortName=COM2
   ```
5. Убедиться, что в **Диспетчере устройств** в разделе «Порты (COM и LPT)» появились `COM1` и `COM2`.

---

### 3.3. Запуск программы

На каждом компьютере из каталога программы запустите файл `KR_KSN.exe`.

При этом на экране появится главное окно программы, а поверх него — диалоговое окно выбора роли.

---

После запуска программы открывается окно выбора роли:

```

│   Вариант 11 — [7,4]-код Хэмминга          │
```

Пользователь должен выбрать одну из двух ролей:

- **Источник (Sender, COM1)** — этот компьютер будет отправлять файл. Программа откроет порт COM1.
- **Приёмник (Receiver, COM2)** — этот компьютер будет принимать файл. Программа откроет порт COM2.

> **Важно:** сначала необходимо запустить программу и выбрать роль **Приёмник**, затем — запустить вторую копию программы и выбрать роль **Источник**.

Кнопка **«Закрыть»** завершает работу программы.

### 3.6. Установка соединения

После запуска программы на обоих компьютерах необходимо установить соединение:

1. На компьютере-**Приёмнике** нажать кнопку **«Подключить»**. Программа откроет порт COM2 и перейдёт в режим ожидания входящего соединения.

2. На компьютере-**Источнике** нажать кнопку **«Подключить»**. Программа откроет порт COM1 и отправит кадр `UPLINK`.

3. Приёмник автоматически ответит кадром `ACK_UPLINK`.

4. В строке состояния обоих окон появится надпись **«Подключено»** зелёного цвета.

В журнале будет отображено:
```
[13:01:05] Порт COM1 открыт. Отправка UPLINK...
[13:01:05] → UPLINK
[13:01:05] ← ACK_UPLINK
[13:01:05] Соединение установлено.
```

Если порт недоступен или занят другой программой, будет выведено сообщение об ошибке:

> «Не удалось открыть порт COM1: Access to the port 'COM1' is denied.»

В этом случае необходимо закрыть другие программы, использующие данный COM-порт, и повторить попытку.

---

### 3.7. Отправка файла (Источник)

После установки соединения на компьютере-Источнике:

1. Нажать кнопку **«Обзор...»** и выбрать текстовый файл для отправки.
   - Содержимое файла немедленно отобразится в панели «Просмотр содержимого файла».
   - В журнале появится сообщение: `Файл выбран: имя_файла.txt (N байт)`.

2. При необходимости задать темп передачи:
   - Выбрать в списке «Темп передачи»: **Медленно (500 мс)**, **Нормально (100 мс)** или **Быстро (10 мс)**.
   - Нажать кнопку **«Задать темп»** — кадр управления темпом будет отправлен Приёмнику.

3. Нажать кнопку **«Отправить файл»**.

Процесс передачи:
- Программа отправляет кадр `FILE_INFO` с именем файла и количеством блоков.
- Приёмник подтверждает готовность кадром `ACK_FILE_INFO`.
- Файл разбивается на блоки по 128 байт. Каждый блок кодируется [7,4]-кодом Хэмминга и отправляется кадром `CHUNK`.
- Приёмник подтверждает каждый блок кадром `ACK_CHUNK`. При ошибке декодирования отправляется `RET_CHUNK` и блок повторяется (до 3 попыток).
- По завершении отправляется кадр `TRANSFER_DONE`.
- Прогресс-бар показывает процент отправленных блоков.

По завершении передачи появится сообщение: **«Файл успешно отправлен!»**

В журнале будет отображено:
```
[13:01:10] Отправка FILE_INFO: "example.txt", 5 блоков...
[13:01:10] → FILE_INFO "example.txt" (5 chunks)
[13:01:10] ← ACK_FILE_INFO
[13:01:10] → CHUNK #0 (128 bytes, attempt 1)
[13:01:10] ← ACK_CHUNK #0
[13:01:11] → CHUNK #1 (128 bytes, attempt 1)
[13:01:11] ← ACK_CHUNK #1
...
[13:01:12] → TRANSFER_DONE
[13:01:12] ← ACK_DONE
[13:01:12] Файл отправлен полностью.
```

### 3.8. Приём файла (Приёмник)

После установки соединения компьютер-Приёмник автоматически ожидает входящий файл.

При получении кадра `FILE_INFO`:
- В поле «Принятый файл» отображается имя файла.
- Прогресс-бар сбрасывается в 0.
- Панель «Просмотр содержимого файла» очищается.

По мере поступления блоков данных:
- Содержимое файла отображается в панели «Просмотр содержимого файла» в **реальном времени**.
- Прогресс-бар заполняется по мере приёма блоков.

По завершении приёма появится сообщение: **«Файл "имя_файла" успешно принят! Нажмите «Сохранить» для записи на диск.»**

Для сохранения файла:
1. Нажать кнопку **«Сохранить»**.
2. В диалоге сохранения выбрать папку и имя файла.
3. Нажать **«Сохранить»**.

---

### 3.9. Управление темпом передачи (прикладной уровень)

Управление темпом передачи реализовано на **прикладном уровне** и доступно с обеих сторон в любой момент во время передачи.

**Со стороны Источника:**
1. Выбрать нужный темп в списке «Темп передачи».
2. Нажать **«Задать темп»**.
3. Кадр `PACE_SLOW` / `PACE_NORMAL` / `PACE_FAST` отправляется Приёмнику.
4. Источник немедленно применяет новую задержку между блоками.

**Со стороны Приёмника:**
1. Выбрать нужный темп в списке «Управление темпом».
2. Нажать **«Задать темп»**.
3. Кадр `PACE_SLOW` / `PACE_NORMAL` / `PACE_FAST` отправляется Источнику.
4. Источник получает кадр и меняет задержку между блоками.

Значения темпа:

| Темп              | Задержка между блоками |
|-------------------|------------------------|
| Медленно          | 500 мс                 |
| Нормально         | 100 мс                 |
| Быстро            | 10 мс                  |

### 3.10. Отключение

Для завершения сеанса связи нажать кнопку **«Отключить»** на любом из компьютеров:
- Будет отправлен кадр `DOWNLINK`.
- Другая сторона ответит кадром `ACK_DOWNLINK`.
- COM-порт закроется.
- Статус изменится на **«Не подключено»** (красный цвет).
- Кнопка «Подключить» станет активной для повторного соединения.

## 4. Устранение неполадок

### 4.1. Сообщения об ошибках

| Сообщение | Причина | Решение |
|-----------|---------|---------|
| «Не удалось открыть порт COM1/COM2» | Порт занят другой программой или не существует | Закрыть другие программы, использующие COM-порт; проверить Диспетчер устройств |
| «Нет активного соединения» | Попытка отправить файл без установленного соединения | Нажать «Подключить» и дождаться статуса «Подключено» |
| «Сначала выберите файл» | Попытка отправить файл без его выбора | Нажать «Обзор...» и выбрать файл |
| «Нет принятых данных» | Попытка сохранить файл до завершения приёма | Дождаться завершения передачи |
| «Блок #N не доставлен. Передача прервана» | Блок не был доставлен после 3 попыток | Проверить кабель; уменьшить темп передачи |

### 4.2. Общие рекомендации

Поскольку Windows — не однозадачная система, возможны случаи, когда некоторые программы забирают все ресурсы компьютера, останавливая выполнение всех других программ. В этом случае удалённая машина может «потерять» соединение, так как она не отвечает на кадры по истечении таймаута (3 секунды на блок). Это может случиться также при плохом качестве связи между компьютерами, например, при большой длине кабеля и высоком уровне помех, и при перегрузке буферов COM-портов.

Во всех перечисленных случаях рекомендуется:
- Уменьшить темп передачи (выбрать «Медленно»).
- Проверить физическое подключение кабеля.
- Убедиться, что на обоих компьютерах нет других программ, использующих COM-порты.
- Перезапустить программу и повторить попытку.

В случае возникновения непредвиденных ошибок запускается стандартный обработчик ошибок Windows.

## 5. Защита данных кодом Хэмминга [7,4]

При передаче каждый байт файла разбивается на два 4-битных полубайта. Каждый полубайт кодируется по схеме [7,4]-кода Хэмминга: к 4 информационным битам добавляются 3 контрольных бита, образуя 7-битное кодовое слово. Таким образом, каждый байт файла передаётся в виде 2 байт (14 информационных бит + 2 нулевых бита).

При декодировании вычисляется синдром ошибки (3 бита). Если синдром не равен нулю, **одиночная ошибка исправляется автоматически** без запроса повтора блока. При двойной ошибке блок запрашивается повторно (кадр `RET_CHUNK`).

Это обеспечивает надёжную передачу данных по линии связи с одиночными битовыми ошибками.
