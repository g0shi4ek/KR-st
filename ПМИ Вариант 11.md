## 1. Наименование

Наименование: **«Программа передачи текстовых файлов»**.

---

## 2. Назначение разработки

Программа должна реализовывать функцию передачи текстовых файлов между двумя ПЭВМ, соединёнными через интерфейс RS-232C нульмодемным кабелем. Программа обеспечивает просмотр содержимого файла при приёме и управление темпом передачи на прикладном уровне. Скорость обмена и параметры COM-порта заданы по умолчанию. Файл задаётся источником. При передаче файла информация защищается [7,4]-кодом Хэмминга с исправлением одиночных ошибок.

---

## 3. Последовательность испытаний

1. Пункты 1–4 необходимо выполнять последовательно.

2. Перед выполнением пункта 6 необходимо выполнить пункты 4 и 5.

3. Перед выполнением пунктов 7–11 необходимо выполнить пункт 6.

---

## 4. Методика испытаний

**А** — пользователь-Источник (Sender, COM1).  
**В** — пользователь-Приёмник (Receiver, COM2).

---

| № | Проверяемая функция | Действия пользователя | Ожидаемый результат |
|---|--------------------|-----------------------|---------------------|
| 1 | Установка и настройка программы эмуляции COM-портов | Загрузить из интернета программу **com0com**, установить её и запустить. Открыть **Setup Command Prompt** из меню «Пуск» → com0com. Выполнить команду: `install PortName=COM1 PortName=COM2` | На компьютере станут доступны 2 виртуальных COM-порта (COM1 и COM2) для работы приложения. В Диспетчере устройств в разделе «Порты (COM и LPT)» появятся COM1 и COM2. |
| 2 | Запуск программы и вывод окна выбора роли | Для **А** и **В**: запустить программу `KR_KSN.exe` на каждом компьютере (или два экземпляра на одном ПК) | Откроется главное окно программы, поверх которого появится диалоговое окно выбора роли с кнопками «Источник (Sender, COM1)», «Приёмник (Receiver, COM2)» и «Закрыть». |
| 3 | Выбор роли Приёмника | Для **В**: нажать кнопку **«Приёмник (Receiver, COM2)»** | Окно выбора роли закроется. В главном окне отобразится: «Роль: Приёмник (Receiver)», «Порт: COM2 \| 9600 bps, 8N1». Станет видна панель «Приёмник — приём файла». |
| 4 | Выбор роли Источника | Для **А**: нажать кнопку **«Источник (Sender, COM1)»** | Окно выбора роли закроется. В главном окне отобразится: «Роль: Источник (Sender)», «Порт: COM1 \| 9600 bps, 8N1». Станет видна панель «Источник — отправка файла». |
| 5 | Подключение Приёмника | Для **В**: нажать кнопку **«Подключить»** | Порт COM2 откроется. В журнале появится: `Порт COM2 открыт. Отправка UPLINK...`. Кнопка «Подключить» станет неактивной, кнопка «Отключить» — активной. Статус: «Не подключено» (ожидание Источника). |
| 6 | Установка соединения (Источник) | Для **А**: нажать кнопку **«Подключить»** | Порт COM1 откроется. Программа отправит кадр `UPLINK`. Приёмник автоматически ответит `ACK_UPLINK`. В журнале обоих окон появятся записи: `→ UPLINK`, `← ACK_UPLINK`, `Соединение установлено.` Статус в обоих окнах изменится на **«Подключено»** (зелёный цвет). Кнопка «Отправить файл» у Источника станет активной после выбора файла. |
| 7 | Выбор файла для отправки | Для **А**: нажать кнопку **«Обзор...»** и выбрать текстовый файл (например, `example.txt`) | В поле пути отобразится полный путь к файлу. В панели «Просмотр содержимого файла» отобразится содержимое выбранного файла. В журнале появится: `Файл выбран: example.txt (N байт)`. Кнопка «Отправить файл» станет активной. |
| 8 | Отправка файла | Для **А**: нажать кнопку **«Отправить файл»** | Начнётся передача файла. В журнале появятся записи: `→ FILE_INFO "example.txt" (N chunks)`, `← ACK_FILE_INFO`, `→ CHUNK #0 (128 bytes, attempt 1)`, `← ACK_CHUNK #0`, и т.д. Прогресс-бар у Источника будет заполняться по мере отправки блоков. |
| 9 | Просмотр содержимого файла при приёме | Для **В**: наблюдать за панелью «Просмотр содержимого файла» во время передачи | В поле «Принятый файл» отобразится имя файла. Содержимое файла будет появляться в панели «Просмотр содержимого файла» в **реальном времени** по мере поступления блоков. Прогресс-бар у Приёмника будет заполняться. |
| 10 | Управление темпом передачи со стороны Источника | Для **А**: во время передачи выбрать в списке «Темп передачи» значение **«Медленно (500 мс)»** и нажать **«Задать темп»** | В журнале Источника появится: `→ PACE_SLOW`, `Темп задан: Slow (500 мс/блок)`. В журнале Приёмника появится: `← PACE_SLOW`, `Темп → SLOW (500 мс)`. Скорость передачи блоков заметно снизится. |
| 11 | Управление темпом передачи со стороны Приёмника | Для **В**: во время передачи выбрать в списке «Управление темпом» значение **«Быстро (10 мс)»** и нажать **«Задать темп»** | В журнале Приёмника появится: `→ PACE_FAST`, `Запрос темпа: Fast`. В журнале Источника появится: `← PACE_FAST`, `Темп → FAST (10 мс)`. Скорость передачи блоков увеличится. |
| 12 | Завершение передачи | Для **А** и **В**: дождаться завершения передачи | После отправки последнего блока Источник отправит кадр `TRANSFER_DONE`. Приёмник ответит `ACK_DONE`. В обоих окнах прогресс-бар достигнет 100%. У Источника появится сообщение: «Файл успешно отправлен!». У Приёмника появится сообщение: «Файл "example.txt" успешно принят! Нажмите «Сохранить» для записи на диск.» Кнопка «Сохранить» у Приёмника станет активной. |
| 13 | Сохранение принятого файла | Для **В**: нажать кнопку **«Сохранить»**, в диалоге выбрать папку и имя файла, нажать «Сохранить» | Откроется стандартный диалог сохранения файла. После подтверждения файл будет записан на диск. В журнале появится: `Файл сохранён: путь\к\файлу.txt`. Появится сообщение: «Файл сохранён: путь\к\файлу.txt». |
| 14 | Вызов справки | Для **А**: выбрать пункт меню **«Справка»** | Откроется окно «Разработчики» с именами авторов: Якимова Т.С., Удалова В.А., Канаева Д.Ч., ИУ5-65Б, 2025 г. |
| 15 | Разрыв соединения | Для **А** или **В**: нажать кнопку **«Отключить»** | Будет отправлен кадр `DOWNLINK`. Другая сторона ответит `ACK_DOWNLINK`. COM-порт закроется. Статус изменится на **«Не подключено»** (красный цвет). Кнопка «Подключить» станет активной, кнопка «Отключить» — неактивной. В журнале появится: `Соединение закрыто.` |
| 16 | Закрытие программы | Для **А** и **В**: нажать на крестик, закрывающий окно | Окно программы будет закрыто. |
